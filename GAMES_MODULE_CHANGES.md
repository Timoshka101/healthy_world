# Обновление модуля Games

## Что изменилось

Раздел `games` перестроен так, чтобы на странице `/games/` основной игрой стала
**«Змейка про здоровое питание»**, а две уже существующие игры были вынесены в
отдельный блок **«Интерактивные тесты»**.

Теперь структура раздела выглядит так:

1. Главная страница `Игры`
   Основной экран со `Змейкой`, выбором уровней, историей попыток и прогрессом БЖУ.
2. Подраздел `Интерактивные тесты`
   Ссылки на старые игры:
   - `Собери здоровое меню`
   - `Подсчитай калории`
3. Страница статистики
   Общая статистика пользователя по всем играм раздела.

## Архитектурное решение

Новая игра не вынесена в отдельное Django-приложение. Она реализована внутри уже
существующего приложения `games`, потому что:

- логика игрового раздела уже сосредоточена в одном месте;
- сохраняется единая навигация `/games/`;
- статистика и история пользователя не дублируются между приложениями;
- в будущем сюда проще добавить ещё одну игру по тому же шаблону.

Для изоляции кода внутри `games` добавлены отдельные модели, API и frontend-файлы
именно для `Змейки`, без смешивания со старыми тестами.

## Новые backend-сущности

### Модели Django

Добавлены новые модели:

- `SnakeGameLevel`
  Уровни игры `Змейка`, хранятся в таблице PostgreSQL `game_level`.
- `SnakeGameAttempt`
  Попытки прохождения уровня, хранятся в таблице `snake_game_session`.
- `SnakeGameUserStats`
  Агрегированная статистика игрока, хранится в таблице `snake_game_user_stats`.

### Поля уровня

Уровни содержат:

- `code`
- `title`
- `difficulty`
- `meal_type`
- `sort_order`
- `target_calories`
- `min_calories`
- `max_calories`
- `target_protein_percent`
- `min_protein_percent`
- `max_protein_percent`
- `target_fat_percent`
- `min_fat_percent`
- `max_fat_percent`
- `target_carb_percent`
- `min_carb_percent`
- `max_carb_percent`
- `time_limit_seconds`
- `initial_speed`
- `speed_step`
- `is_active`

### Поля результата

Каждая попытка сохраняет:

- пользователя;
- уровень;
- время начала и завершения;
- длительность;
- победу или поражение;
- причину поражения;
- итоговые калории;
- итоговые белки, жиры, углеводы;
- итоговые проценты БЖУ;
- очки.

## Начальные уровни

Через миграцию автоматически создаются 3 уровня:

1. `Завтрак`
   - лёгкий
   - 600 ккал
   - 2 минуты
2. `Обед`
   - сложный
   - 900 ккал
   - 3 минуты
3. `Ужин`
   - средний
   - 600 ккал
   - 1.5 минуты

## API для фронтенда

Добавлены JSON-endpoint'ы:

- `GET /api/games/snake/levels/`
  Список уровней
- `GET /api/games/snake/levels/<id>/`
  Детали уровня, конфиг поля и список игровых объектов
- `POST /api/games/snake/result/`
  Сохранение результата прохождения
- `GET /api/games/snake/history/`
  История попыток пользователя

## Frontend игры

Добавлены файлы:

- `static/games/snake_game.js`
- `static/games/snake_game.css`

Реализовано:

- поле змейки на `canvas`;
- выбор уровня на странице `/games/`;
- управление стрелками и `WASD`;
- рост змейки после полезной еды;
- ускорение по мере прогресса;
- случайный спавн полезной и вредной еды;
- эффект потери управления от колбы;
- подсчёт калорий и БЖУ;
- проверка победы и поражения;
- индикаторы прогресса;
- отправка результата в Django API;
- загрузка истории прохождений.

## Изменения в шаблонах проекта

- в `templates/base.html` добавлен блок `extra_head`;
- пункт навигации переименован в `Игры`;
- страница `/games/` теперь показывает `Змейку` как основную игру;
- старые игры перенесены в отдельный визуальный блок `Интерактивные тесты`;
- страница статистики обновлена и теперь показывает результаты по `Змейке`.

## Проверка изменений

Для локальной проверки без поднятого PostgreSQL добавлен файл:

- `healthy_world/test_settings.py`

Он использует SQLite только для тестов и не меняет основную PostgreSQL-конфигурацию
приложения.

Проверенные команды:

```bash
python3 manage.py check --settings=healthy_world.test_settings
python3 manage.py test --settings=healthy_world.test_settings
python3 manage.py makemigrations --check --dry-run games --settings=healthy_world.test_settings
```

## Что осталось прежним

- существующие две игры не удалены;
- старые модели `GameLevel`, `UserGameResult`, `CalorieCounterQuestion` и `Product`
  продолжают работать;
- маршруты старых игр сохранены.
